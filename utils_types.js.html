/**
 * An Item (or asset, in some cases) is a key object in this software. Here is the life cycle of an item:
 * - Item is received off a shipment (using the scanner, but the Web App also has that functionality)
 *   - This action creates an item in the DB, with the current date as `date_received` and all necessary data inputted
 *   - If this is the first item received for a purchase order, sets the purchase order's `date_received` field to the current 
 * - Item is placed either on a [Box]{@link Box} or [Pallet]{@link Pallet}
 *   - All non-desktops are placed into a Box which increases that Box's `item_count`
 *   - Desktops can be placed on a Pallet directly, updating that Pallet's `box_count`
 * - From there, the Item can be moved to other Boxes/Pallets, but will eventually be shipped as part of a Box/Pallet
 * 
 * @typedef Item
 * @property {number} id 
 * @property {number} asset_num
 * @property {number} po_num
 * @property {number} [box_id]
 * @property {number} [pallet_id]
 * @property {string} description
 * @property {string} condition
 * @property {string} date_received
 * @property {string} [date_shipped]
 */

/**
 * A Tray is one of the three containers in this software. Here are its details:
 * - A Tray holds [Tray Items]{@link TrayItem} that are either RAM sticks or CPUs
 * - A Tray will always hold only one type of Tray Item, which is displayed in the `has_ram` flag
 * - A Tray will normally have a set number of Tray Items, those being 25 or 50 RAM sticks or 21 CPUs
 *   - A Tray, though, can be left unfilled and a user can add a CPU to the next unfilled Tray
 * - A Tray is created at the same time as Tray Items in the Inventory screen
 * 
 * @typedef Tray
 * @property {number} id
 * @property {number} item_count
 * @property {number} location_id
 * @property {boolean} has_ram
 * @property {string} date_created
 * @property {string} [date_shipped]
 */

/**
 * A Tray Item represents a number of like CPUs or RAM sticks. The non-optional fields are 
 * available for both a CPU and RAM stick, but optionals are only valid for one or the other.
 * A Tray Item that describes RAM sticks has: 
 * - `brand`
 * - `model`
 * - `form_factor`
 * - `shielded`
 * - `type`
 * - `version`
 * - `speed` (in MHz)
 * - `size`
 * - `quantity` (25 or 50)
 * 
 * While a Tray Item that describes CPUs has:
 * - `type`
 * - `model` (4-5 digit number)
 * - `speed` (in GHz)
 * - `has_imperfections` (surface imperfections)
 * - `has_chipped_edges`
 * - `grade` (A)
 * - `quantity` (generally 1, but could be multiple)
 * 
 * Tray Items are created at the same time as [Trays]{@link Tray} on the Inventory Screen
 * 
 * @typedef TrayItem
 * @property {number} id
 * @property {number} tray_id
 * @property {number} po_num
 * @property {number} speed
 * @property {number} [size]
 * @property {number} quantity
 * @property {boolean} [is_shielded]
 * @property {boolean} [has_imperfections]
 * @property {boolean} [has_chipped_edges]
 * @property {boolean} is_ram
 * @property {string} asset_num
 * @property {string} [brand]
 * @property {string} model
 * @property {TrayType} type
 * @property {string} [grade]
 * @property {string} [date_shipped]
 * @property {string} date_created
 * @property {TrayFormFactor} [form_factor]
 * @property {TrayVersion} [version]
 */

/**
 * A Box is one of three containers that are available in this software. Here is its life cycle:
 * - A Box can be created at any time using the Inventory screen. Doing so generates a new Box ID and
 *      an associated Box Tag to print out. This Box Tag can also be printed via the Reports screen
 * - After receiving [Items]{@link Item}, any number of Items can be placed in the Box
 *   - Moving an Item onto a Box updates the Box's `item_count` and the Item's `box_id`
 * - The Box can be moved from location to location, as well as placed on a [Pallet]{@link Pallet}
 *   - If a Box is moved to a location while it is on a Pallet, the user will be prompted with a confirmation,
 *      then the Box will be taken off the pallet (`pallet_id` set to `null`) and to that location
 *   - If a Box is moved onto a Pallet, the Box's `location_id` is updated to match the Pallet's. Additionally, the
 *      Box's `pallet_id` and the `pallet_id` of all Items in the Box are updated to match the Pallet's.
 * - A Box can be shipped either by itself or on a Pallet
 * 
 * @typedef Box
 * @property {number} id
 * @property {number} [pallet_id]
 * @property {number} [location_id]
 * @property {number} item_count
 * @property {string} description
 * @property {string} date_created
 * @property {string} date_shipped
 */

/**
 * A Pallet is one of three containers that are available in this software. Here is its life cycle:
 * - A Pallet can be created at any time using the Inventory screen. Doing so generates a new Pallet ID and
 *      an associated Pallet Tag to print out. This Pallet Tag can also be printed via the Reports screen
 * - The only things that can go on a Pallet are [Boxes]{@link Box} and "Desktop" [Items]{@link Item}.
 * - Moving an Item or Box onto a Pallet updates that Pallet's `box_count`, as well as updates the Box/Item's `pallet_id`. 
 * - The Pallet can be moved from location to location
 *   - When a Pallet is moved, the Boxes on the Pallet are also updated to show the change.
 * - Either of the above objects can be placed onto a Pallet. Any time the pallet is moved or shipped, the
 *      Box or Item also reflects those changes
 * - A Pallet will eventually be shipped
 * @typedef Pallet
 * @property {number} id
 * @property {number} [location_id]
 * @property {number} box_count
 * @property {string} description
 * @property {string} date_created
 * @property {string} date_shipped
 */

/**
 * A Purchase Order is one of two types of Orders that are present in the software. Here are its details:
 * - A Purchase Order can be created by going to the Purchase Order screen and clicking the + button
 * - A Purchase Order can be edited by going to the Purchase Order screen and clicking the pencil button
 * - A Purchase Order can be either Interco or Non-Interco. If it's an Interco Order, contact information
 *      (for the supplier) is not needed. Furthermore, no [Purchase Order Items]{@link PurchaseOrderItem} are created
 * - If it's a Non-Interco order, contact information is needed, as well as Purchase Order Items, which are what is expected to come on the shipment.
 * - Once an [Item]{@link Item} is received with a valid Purchase Order number, that Purchase Order whose ID matches that number is marked as
 *      received (`date_received` set to that datetime) and the Purchase Order can no longer be edited
 * - A Purchase Order's Items present in the inventory may not always add up to total the expected Purchase Order Items inputted at creation.
 *      Because of this, the user can go to the Order Review tab and see the discrepencies for a Purchase Order
 * @typedef PurchaseOrder
 * @property {number} id
 * @property {boolean} is_itc
 * @property {string} date_created
 * @property {string} [date_received]
 * @property {string} supp_name
 * @property {string} supp_street
 * @property {string} supp_city
 * @property {string} supp_state
 * @property {string} supp_zip
 * @property {string} supp_num
 * @property {string} cust_name
 * @property {string} cust_street
 * @property {string} cust_city
 * @property {string} cust_state
 * @property {string} cust_zip
 * @property {string} [contact_first]
 * @property {string} [contact_last]
 * @property {string} [contact_num]
 * @property {string} [contact_email]
 */

/**
 * An Invoice is one of two types of Orders in this software. Here are its details:
 * - An Invoice can be created by going to the Invoice screen and clicking the + button
 * - An Invoice can be edited by going to the Invoice screen and clicking the pencil button
 * - An Invoice is created with [Pick Ticket Items]{@link PickTicketItem}, which are [Boxes]{@link Box} and [Pallets]{@link Pallet}
 *      that will be included on the outgoing shipment
 * - Once an Invoice is created, the individual [Items]{@link Item} in the order can be viewed (even though the Pick Ticket Items are containers)
 *   - This can also be viewed as a report via the Invoice or Reports screen
 * - The Invoice will also be generated as a Pick Ticket, which can be viewed on the Order Picking screen
 *   - This screen details the Boxes and Pallets needed to be picked and their locations, which can also be viewed as a report
 * @typedef Invoice
 * @property {number} id
 * @property {boolean} picked
 * @property {string} date_created
 * @property {string} [date_shipped]
 * @property {string} supp_name
 * @property {string} supp_street
 * @property {string} supp_city
 * @property {string} supp_state
 * @property {string} supp_zip
 * @property {string} cust_name
 * @property {string} cust_street
 * @property {string} cust_city
 * @property {string} cust_state
 * @property {string} cust_zip
 * @property {string} contact_first
 * @property {string} contact_last
 * @property {string} contact_num
 * @property {string} contact_email
 */
